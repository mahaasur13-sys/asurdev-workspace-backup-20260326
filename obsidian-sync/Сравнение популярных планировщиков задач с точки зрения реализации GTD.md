## 💡 Как выбрать подходящий планировщик

- **Оцените уровень погружения в GTD**  
  - Для начинающих: *Todoist* или *TickTick*.  
  - Для опытных и готовых к настройке: *SingularityApp* или *LeaderTask*.  

- **Определите контекст использования**  
  - Индивидуальная продуктивность: *Todoist*, *TickTick*.  
  - Командная работа: *WEEEK*, *LeaderTask*.  

- **Проверьте критерии на практике**  
  - Используйте бесплатные тарифы.  
  - Обратите внимание на скорость добавления задачи (идея должна фиксироваться за секунды).  
  - Убедитесь в синхронизации между устройствами и удобном интерфейсе.

## 🛠 Практические шаги по настройке GTD

1. **Сбор**  
   - Настройте Inbox для быстрой фиксации мыслей.  
   - Используйте email- или Telegram-ботов, если приложение поддерживает.  

2. **Организация**  
   - Создайте списки/категории:  
     - *Проекты* (многошаговые задачи)  
     - *Следующие действия* (можно метки)  
     - *Когда-нибудь*  
     - *Ожидание*  

3. **Обзор**  
   - Планируйте еженедельный обзор.  
   - Пересматривайте проекты и списки, чтобы система оставалась актуальной.  
   - Некоторые приложения (*SingularityApp*) предлагают встроенные режимы для обзора.

## 🛠 Как настроить GTD в WEEEK самостоятельно

Поскольку готовых материалов мало, вот план, как можно адаптировать метод GTD в WEEEK:

|Шаг|Что сделать в WEEEK|Комментарии|
|---|---|---|
|**1. Inbox / Сбор**|Использовать раздел «Входящие» (или аналоги) для всего, что приходит: идеи, задачи, письма.|Главное — быстро фиксировать всё, чтобы не держать в голове.|
|**2. Проекты**|Создать проекты для мультшаговых задач. В WEEEK можно использовать папки / проекты для этого.|Следить, чтобы у каждого проекта был следующий конкретный шаг.|
|**3. Следующие действия (Next Actions)**|Метки, задачи с ярлыками или отдельный список “Следующие действия” — те, что можно выполнить сразу.|Важно, чтобы не было “зависших” задач без ясных шагов.|
|**4. Ожидание / Delegate**|Списки или теги “Ожидается / Waiting For” для задач, где ждёшь ответа / выполнения кем-то другим.|Позже проверять, продвигаются ли они.|
|**5. Когда-нибудь / Когда-нибудь может быть (Someday/Maybe)**|Раздел или метка для идей, которые пока не актуальны.|Проверять при обзоре.|
|**6. Еженедельный обзор (Weekly Review)**|Раз в неделю пересматривать:||

- входящие → проектные / задачи
    
- календарь прошлой недели и следующей
    
- список проектов: есть ли продвижение, есть ли следующий конкретный шаг
    
- список “Ожидание” и “Когда-нибудь” | Можно поставить напоминание или шаблон для обзора, чтобы не забывать. |
