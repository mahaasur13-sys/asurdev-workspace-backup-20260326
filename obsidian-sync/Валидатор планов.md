# VAL: Валидатор планов PDDL

*VAL (Validator)* — это ключевой программный инструмент для *валидации (проверки)* планов, сгенерированных планировщиками, на соответствие спецификациям домена и задачи на языке [[PDDL]]. Он проверяет, является ли план выполнимым, достигает ли он цели и соответствует ли логике домена.

Разработчики: Мария Фокс, Дерек Лонг и др.

---

### ⚙️ 2. Ключевые особенности

- *Основная функция*: Проверяет, что каждое действие в плане применимо, и что конечная цель достигается.
- *Поддержка расширений [[PDDL]]*:
    - *PDDL 2.2+*: Непрерывные эффекты, выводимые предикаты (`derived predicates`), временны́е начальные литералы (`timed initial literals`).
    - *Числовые и объектные флюенты*: Валидирует планы с математическими расчетами и переменными, значениями которых являются объекты.
- *Отчеты*: Может генерировать детальные отчеты о валидации в формате LaTeX.
- *Кроссплатформенность*: Доступны бинарные файлы для Linux, Windows и macOS.

### 🔄 3. Место в рабочем процессе

VAL используется *после* того, как [[Планирование в ИИ|планировщик]] (например, [[Fast Downward]] или OPTIC) сгенерировал план.

1. *Создание файлов*: Пишутся файлы `domain.pddl` и `problem.pddl`.
2. *Генерация плана*: Планировщик создает файл с решением (например, `plan.ipc`).
3. *Валидация*: VAL проверяет корректность плана, анализируя:
    - *Предусловия* (preconditions)
    - *Эффекты* (effects)
    - *Достижение цели* (goal achievement)
4. *Отладка*: Если план невалиден, вывод VAL помогает найти ошибку в домене, задаче или самом плане.

### 🛠️ 4. Использование VAL

#### Установка
Бинарные файлы можно скачать с [официального репозитория на GitHub](https://github.com/KCL-Planning/VAL).

#### Базовый синтаксис
VAL запускается из командной строки:
- *Успех*: Выводит `Plan valid` и стоимость плана.
- *Ошибка*: Указывает на конкретное действие и причину сбоя (например, невыполненное предусловие).

### ⚠️ 5. Ограничения и альтернативы

#### Ограничения:
- *Временны́е планы*: Не полностью поддерживает планирование с действиями, имеющими длительность (`durative actions` из PDDL 2.1+).
- *Ошибки*: Как и любое ПО, может содержать баги. Интерпретация семантики PDDL может отличаться от интерпретации планировщиков.

#### Альтернативы:
- *INVAL*: Более простой валидатор для перекрестной проверки.
- *UPV (Universal Planning Validator)*: Инструмент в разработке для поддержки классического, временнóго и многоагентного планирования.

### 💻 6. Интеграция с инструментами

- *VS Code*: [Расширение PDDL для VS Code](https://marketplace.visualstudio.com/items?itemName=jan-dolejsi.pddl) интегрирует VAL для валидации планов прямо в редакторе.
- *Planning.Domains*: Онлайн-сервис использует VAL в своем API для валидации планов.
