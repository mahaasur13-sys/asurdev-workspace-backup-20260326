# Обзор локальных языковых моделей (LLM)

Этот документ содержит сводную информацию о различных локальных и открытых языковых моделях, которые можно использовать с такими инструментами, как [Ollama](https://ollama.com/) и плагином [BMO Chatbot](https://github.com/longy2k/obsidian-bmo-chatbot) в Obsidian.

## Как использовать эти модели?

1. Установите Ollama: Скачайте и установите Ollama с официального сайта.
2. Скачайте модель: Откройте терминал (командную строку) и выполните команду `ollama run <имя_модели>`. Например:
        ollama run qwen3-4b-2507
    
3. Настройте BMO Chatbot: В настройках плагина BMO в Obsidian выберите провайдера `Ollama` и впишите имя скачанной модели в поле `Model`.

## Ключевые термины

-   B (Billion): Миллиард параметров. Чем больше параметров, тем, как правило, "умнее" и "знающее" модель, но и тем больше ресурсов она требует.
-   MoE (Mixture of Experts): Архитектура, при которой модель состоит из множества "экспертов" (нейросетей поменьше). Для каждого запроса активируется только небольшая часть этих экспертов, что делает большие модели очень быстрыми и эффективными.
-   Активные параметры: Количество параметров, которое используется в MoE-модели для обработки одного запроса.

✏️ 1. Общие принципы адаптации моделей

Адаптация модели — это процесс её специализации для повышения эффективности в конкретной области. Это позволяет модели понимать специфическую терминологию, следовать нужным форматам и давать более точные и релевантрые ответы.

📚 Основные подходы:

🔖 1.1. Файн-тюнинг (Fine-tuning)
Это дообучение предварительно обученной модели на вашем собственном, более узкоспециализированном наборе данных.

-   Как это работает: Вы "показываете" модели примеры правильных ответов на вопросы из вашей области. Модель корректирует свои внутренние веса, чтобы лучше соответствовать этим примерам.
-   Когда использовать: Когда требуется глубокое понимание специфического стиля, терминологии или формата данных.
-   Пример датасета: Файл с сотнями или тысячами пар `{"prompt": "вопрос из вашей области", "completion": "идеальный ответ"}`.

🔖 1.2. Методы Parameter-Efficient Fine-Tuning (PEFT)
Современные подходы, которые не требуют переобучения всей модели, что экономит ресурсы.

-   Prompt/Prefix Tuning: К каждому запросу добавляется небольшой обучаемый "префикс" (вектор), который направляет модель к нужному ответу, не изменяя её основные веса.
-   LoRA/QLoRA (Low-Rank Adaptation): Обучаются только небольшие дополнительные матрицы, которые "встраиваются" в слои модели. Это самый популярный и эффективный метод.

🔖 1.3. Расширение контекста с помощью RAG (Retrieval-Augmented Generation)
Интеграция модели с внешней базой знаний (документы, статьи, базы данных).

-   Как это работает: Перед тем как модель отвечает на вопрос, специальная система находит в базе знаний наиболее релевантную информацию и добавляет её в контекст запроса. Модель генерирует ответ, основываясь на этой свежей информации.
-   Когда использовать: Когда модели нужны актуальные или частные данные, которых не было в её первоначальном обучении (например, истории болезни, финансовые отчеты за последний квартал).

✏️ 2. Адаптация под конкретные сценарии

📚 🩺 Телемедицина

Цель: Создание ассистента для врачей: анализ симптомов, предварительная диагностика, расшифровка анализов и суммаризация историй болезни.

Модели для использования:
-   `qwen3-4b-thinking-2507`: Для сложных рассуждений и анализа неоднозначных случаев.
-   `hermes-4-70b`: Для генерации качественных и структурированных медицинских текстов.
-   `ernie-4.5-21b-a3b`: Хороший баланс для работы с медицинской терминологией.

Этапы адаптации:
1. Подготовка датасета:
    -   Собрать анонимизированные данные: истории болезни, протоколы лечения, научные статьи (PubMed, MedLine), результаты анализов.
    -   Важно: Полностью удалить всю персональную информацию пациентов (ФИО, адреса, даты).
    -   Структурировать данные в формате "вопрос-ответ" или "симптом-диагноз".
2. Выбор метода адаптации:
    -   Файн-тюнинг (LoRA): Обучить модель на датасете для понимания медицинской лексики и логики постановки диагноза.
    -   RAG: Создать векторную базу данных из медицинских справочников и протоколов. Это позволит модели давать ответы на основе актуальных клинических рекомендаций.
3. Интеграция и безопасность:
    -   Интегрировать модель с медицинскими информационными системами (МИС) через защищенный API.
    -


Обязательно: Внедрить систему верификации. Ответы модели должны проходить проверку квалифицированным врачом перед тем, как будут использованы для принятия решений.

📚 📈 Анализ финансовых данных

Цель: Прогнозирование рыночных трендов, анализ рисков, автоматизация отчетности, выявление мошеннических транзакций.

Модели для использования:
-   `qwen3-4b`: Для быстрой обработки больших объемов текстовых и числовых данных.
-   `devstral-small-2507`: Если задача связана с написанием скриптов для анализа данных (например, на Python).
-   `lfm2-1.2b`: Для встраиваемых решений с низкой задержкой (например, скоринг транзакций в реальном времени).

Этапы адаптации:
1. Подготовка датасета:
    -   Использовать исторические данные о котировках, финансовые отчеты компаний (SEC filings), экономические новости (Bloomberg, Reuters), данные о транзакциях.
2. Выбор метода адаптации:
    -   Файн-тюнинг: Обучить модель на финансовых отчетах для извлечения ключевых показателей (выручка, EBITDA, долг).
    -   RAG: Подключить модель к API биржевых платформ и новостных агрегаторов для анализа данных в реальном времени.
    -   Гибридный подход: Использовать традиционные модели прогнозирования (LSTM, GRU) для числовых рядов, а LLM — для анализа текстовой информации (новостной фон, отчеты).

📚 🤖 Робототехника

Цель: Интерпретация голосовых или текстовых команд, планирование последовательности действий, анализ данных с сенсоров, принятие решений в динамической среде.

Модели для использования:
-   `qwen3-4b-thinking-2507`: Для сложных задач планирования и принятия решений в условиях неопределенности.
-   `lfm2-1.2b`: Для быстрой реакции и работы на бортовых (edge) устройствах с ограниченными ресурсами.

Этапы адаптации:
1. Подготовка датасета:
    -   Данные из симуляций (например, в Gazebo): логи команд, состояния сенсоров, успешные/неуспешные траектории движения.
    -   Текстовые инструкции и соответствующие им последовательности действий робота.
2. Выбор метода адаптации:
    -   Файн-тюнинг: Обучить модель преобразовывать команды на естественном языке ("принеси мне яблоко со стола") в исполняемый код или последовательность команд для контроллера робота (например, через ROS).
    -   Интеграция: Связать LLM с системами компьютерного зрения (для распознавания объектов) и контроллерами движения. Модель выступает в роли "мозга", который ставит цели для других подсистем.

✏️ 3. Ограничения и этические аспекты

> [!WARNING]
> Игнорирование этих аспектов может привести к серьезным финансовым, репутационным и юридическим последствиям.

-   Телемедицина:
    -   Риск: Ошибка в диагнозе или рекомендации.
    -   Решение: Выводы модели должны иметь статус "рекомендации для врача", а не "окончательного диагноза". Обязательна верификация человеком.
-   Финансы:
    -   Риск: Неправильные прогнозы могут привести к убыткам. Модели склонны к "галлюцинациям" и не могут предсказывать "черных лебедей".
    -   Решение: Использовать LLM как один из инструментов анализа, а не единственный. Внедрять модули риск-менеджмента и стоп-лоссы.
-   Робототехника:
    -   Риск: Некорректное действие робота может привести к физическому ущербу.
    -   Решение: Внедрять многоуровневые системы безопасности, включая физические "красные кнопки" и программные ограничители, которые не зависят от LLM.

Успешная интеграция LLM в прикладные системы — это не только выбор мощной модели, но и тщательная работа с данными, правильный выбор метода адаптации и создание надежной, безопасной инфраструктуры.
