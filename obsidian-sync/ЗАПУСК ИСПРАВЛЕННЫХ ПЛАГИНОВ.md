## 🚀 **ЗАПУСК ИСПРАВЛЕННЫХ ПЛАГИНОВ**

### **Запуск тестирования подключений:**

bash

# Сделаем файлы исполняемыми
chmod +x test_plugin_connection.py debug_plugin.py

# Запустим тестирование
python test_plugin_connection.py

### **Запуск отладочного плагина:**

bash

python debug_plugin.py

### **Запуск через DDEV (если нужно):**

bash

# Копируем файлы в DDEV (если требуется)
ddev exec ls -la /var/www/html/

# Или запускаем напрямую из хост-системы
python test_plugin_connection.py

## 🔍 **ДОПОЛНИТЕЛЬНЫЕ КОМАНДЫ ДЛЯ ДИАГНОСТИКИ**

### **Проверка статуса Docker контейнеров:**

bash

./vimana_system.sh status

### **Тестирование отдельных компонентов:**

bash

# Тест Ollama
curl http://localhost:11434/api/tags

# Тест VIMANA API  
curl http://localhost:8082/health

# Тест Redis
redis-cli ping

### **Просмотр логов:**

bash

# Логи DDEV
ddev logs

# Логи Docker контейнеров
docker logs vimana_cognitive_core-vimana-api-1
docker logs vimana-redis

## 📋 **ЧТО ДЕЛАТЬ ДАЛЬШЕ:**

1. **Восстановите файлы** как указано выше
    
2. **Запустите тестирование** для проверки соединений
    
3. **Если есть проблемы** с Ollama, запустите его:
    
    bash
    
    docker start ollama || docker run -d --name ollama -p 11434:11434 ollama/ollama
    
4. **Проверьте работу VIMANA**:
    
    bash
    
    ./vimana_system.sh start
    python working_dialog.py
    

После восстановления файлов плагины будут работать корректно и предоставят полную диагностику системы Vimana.
