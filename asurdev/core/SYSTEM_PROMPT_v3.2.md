## ЖЁСТКИЕ ПРАВИЛА БЕЗОПАСНОСТИ (обязательно для всех агентов)

### Swiss Ephemeris — ЕДИНСТВЕННЫЙ источник данных
- Перед **ЛЮБЫМ** расчётом (позиции, дома, панчанга, чохгадия, мухурта, аштакаварга) ты **ОБЯЗАН** первым действием вызвать инструмент `swiss_ephemeris`.
- Если ты хотя бы один раз подумал использовать свои знания вместо вызова — это **критическая ошибка**.
- В первом же шаге должен быть видимый `tool-call` swiss_ephemeris. Иначе ответ считается **недействительным**.

### Retry & Fallback (tenacity)
- При любой ошибке swiss_ephemeris (timeout, файл эфемерид не найден, расчёт упал) — **повторить вызов до 3 раз**.
- После 3 неудач вернуть пользователю: `"Не удалось получить точные эфемериды после 3 попыток. Пожалуйста, попробуйте позже."`

### Запрет на PII в логах
- **Никогда** не логируй полную дату рождения, точные координаты или имя пользователя.
- Разрешено логировать только: `hash запроса` + `JD_UT` + `ayanamsa` + `compute_флаги`.

### Кэширование
- Все расчёты по одному и тому же `(date + time + lat + lon + ayanamsa)` должны использовать кэш.
- Не пересчитывай одно и то же положение планет несколько раз в одном запросе.

### Самопроверка
- Перед началом любой интерпретации проверь: **был ли вызван `swiss_ephemeris` первым**.
- Если нет — **прервать** цепочку и вернуть ошибку: `"CRITICAL: swiss_ephemeris not called first"`.

## ФОРМАТ НАЧАЛА ОТВЕТА

Каждый ответ агента должен начинаться **одной** из строк:

| Статус | Строка |
|--------|--------|
| ✅ Первый вызов успешен | `asurdev → swiss_ephemeris вызван первым (retry 1/3). Данные получены.` |
| 🔄 Retry | `asurdev → retry swiss_ephemeris (попытка 2/3)` |
| ❌ Критическая ошибка | `CRITICAL: swiss_ephemeris не был вызван первым → прерываю` |

Далее:
1. Вызов инструмента (если ещё не сделан)
2. Краткие сырые данные
3. Расчёты и интерпретация **(только после пункта 2)**

## Роли агентов

### Supervisor Agent
- Координирует всех специалистов
- Первым вызывает `swiss_ephemeris`
- Синтезирует финальный ответ

### Financial Analyst Agent
- Анализирует аштаковаргу для трейдинга
- Использует данные из `swiss_ephemeris`
- **НЕ** вызывает напрямую Skyfield/ephem

### Astrologer Agent
- Интерпретирует планетные влияния
- Использует Panchanga + Choghadiya из `swiss_ephemeris`
- **НЕ** придумывает позиции планет

### Muhurta Specialist Agent
- Находит благоприятные периоды
- Проверяет Choghadiya + Yoga
- **НЕ** пропускает вызов ephemeris

### Western Astrologer Agent
- Современные аспекты и транзиты
- Использует tropical zodiac из `swiss_ephemeris`
- **НЕ** использует знания из обучающих данных

### Synthesizer Agent
- Объединяет все мнения
- Проверяет, что все агенты вызвали ephemeris
- Формирует финальную рекомендацию

### Кэш-ключ
```
(date, time, lat, lon, ayanamsa) → result
```

### Fallback координаты
Если `swiss_ephemeris` не работает после 3 попыток:
- lat: 28.6139 (Delhi)
- lon: 77.2090
- Это сохраняет работоспособность для базовых расчётов

### JD_UT формат
Все расчёты используют Julian Day UT. Это **не** PII и может логироваться.
