## 🏢 Как в офисе

Представь финансовую компанию с 7 аналитиками, 1 секретаршей и 1 директором.

```
                 ГЕНЕРАЛЬНЫЙ ДИРЕКТОР
                    (Orchestrator)
                   Принимает запросы
                   Распределяет задачи
                   Собирает отчёты
                   Выдаёт решения
                         │
     ┌───────────────────┼───────────────────┐
     │                   │                   │
     ▼                   ▼                   ▼
┌─────────┐       ┌─────────┐       ┌─────────┐
│ Market  │       │  Bull   │       │  Bear   │
│Analyst  │       │Researcher│       │Researcher│
└────┬────┘       └────┬────┘       └────┬────┘
     │                   │                   │
     └───────────────────┼───────────────────┘
                         │
                         ▼
              ┌─────────────────────┐
              │   СОВЕТ ДИРЕКТОРОВ  │
              │    (Synthesizer)    │
              │  Собирает мнения    │
              │  Считает голоса      │
              │  Выдаёт вердикт      │
              └─────────────────────┘
```

## 👔 Orchestrator — Генеральный Директор

### Кто это?
Это "мозг" системы. Как генеральный директор в компании.

### Что он делает (функции):
```
1. ПРИЁМ ЗАПРОСОВ — "Клиент хочет мнение по BTC"
2. ПОДГОТОВКА ДАННЫХ — "Получить свежие данные"
3. РАСПРЕДЕЛЕНИЕ ЗАДАЧ — "Аналитикам: работаем!"
4. КОНТРОЛЬ ИСПОЛНЕНИЯ — "Ждём пока все выполнят"
5. СБОРКА ОТЧЁТОВ — "Все сдали? Собираем"
6. ВЫДАЧА РЕШЕНИЯ — "Вот наше мнение"
```

### Что ОН НЕ делает:
```
❌ Он НЕ анализирует сам
❌ Он НЕ торгует
❌ Он НЕ даёт финансовых советов
```

### Как он разговаривает:
```
ОР: "Market Analyst! 120 секунд. BTC=$67K, RSI=58. Давай!"
МА: "Понял! Начинаю..."
ОР: (через 30 сек) "Bull! Твоя очередь."
ОР: (через 60 сек) "Synthesizer, собери всё!"
```

## 📊 Market Analyst — Главный Технический Аналитик

### Его специализация:
```
• Ценовые паттерны (свечи)
• Тренды (вверх/вниз/вбок)
• Индикаторы (RSI, MACD, MA)
• Поддержка и сопротивление
• Объёмы
```

### Его ответ:
```
"Восходящий тренд. Цена выше MA200.
 RSI=58 (не перекуплен). Объёмы растут.
 Сигнал: BUY. Уверенность: 75%."
```

### Его права:
```
✅ Может давать ЛЮБУЮ рекомендацию
✅ Может запросить дополнительные данные
✅ Может сказать "данных недостаточно"
```

### Его ограничения:
```
❌ НЕ может торговать
❌ НЕ видит что думают другие
❌ НЕ знает что будет завтра
❌ НЕ учитывает новости
```

## 🐂 Bull Researcher — Бычий Исследователь

### Его специализация:
```
• Фундаментальные факторы роста
• Партнёрства и релизы
• Whale activity
• ETF и институциональные потоки
• Он-чейн метрики
```

### Его задача:
Найти ВСЕ возможные причины роста. Даже если их мало.

### Он НЕ такой как Market Analyst:
```
Market Analyst: "Цена упала на 5%. Смотрю ТЕХНИКУ."
Bull Researcher: "Цена упала, НО! Whale купили 10,000 BTC.
                  Это бычий сигнал!"
```

---

## 🐻 Bear Researcher — Медвежий Исследователь

### Его специализация:
```
• Риски и угрозы
• Регуляторные проблемы
• Whale selling
• Макро факторы
• Корреляции
```

### Его задача:
Найти ВСЕ возможные причины падения. Даже если их мало.

### Он НЕ такой как Bull Researcher:
```
Bull Researcher: "BTC вырос на 10%! Отличные новости!"
Bear Researcher: "Да, вырос, НО whale продали 5000 BTC.
                  Это медвежий сигнал!"
```

## ⚖️ Synthesizer — Совет Директоров

### Кто это?
"Совет Директоров" — группа которая смотрит все отчёты и принимает коллективное решение.

### Как он считает голоса:
```
ГОЛОСОВАНИЕ:

Market Analyst:      BUY (75%)  → 75 очков
Bull Researcher:     BUY (80%)  → 80 очков  
Bear Researcher:     SELL(60%)  → 60 очков
SMC Agent:           BUY (70%)  → 70 очков
Monte Carlo:         HOLD(50%)  → 50 очков
Gann Agent:         SELL(65%)  → 65 очков
Dow Theory:          BUY (72%)  → 72 очков

ИТОГО:
BUY очки:   75+80+70+72 = 297
SELL очки:  60+65 = 125
HOLD очки:  50

297 > 125 → ПОБЕДА ЗА BUY
```

### Его формат — C.L.E.A.R.:
```
C - CONTEXT    (Что происходит)
L - LOGIC      (Почему мы так решили)
E - EVIDENCE   (Что говорят цифры)
A - ASSESSMENT (Какие риски)
R - RECOMMENDATION (Что делать)
```

## 🤖 Автоматические агенты (без LLM)

### SMC Agent — Следопыт Умных Денег
```
Что делает:
• Находит Order Blocks (зоны где покупают большие)
• Находит Fair Value Gaps (дыры на графике)
• Находит Liquidity Pools (зоны выноса стопов)
```

### Monte Carlo — Статист
```
Что делает:
• Запускает 10,000 симуляций
• Считает вероятности
• Даёт диапазоны (пессимизм - медиана - оптимизм)

Аналогия: "Завтра 15-25 градусов" (не точную температуру)
```

### Gann Agent — Математический Волшебник
```
Что делает:
• Square of 9 (ценовые уровни)
• Death Zones (даты "смерти")
• Углы Ганна

❌ ТОЛЬКО математика, НЕ новости и фундаментал
```

### Dow Theory — Классик
```
Что делает:
• Определяет тренд по правилам 1900 года
• Сравнивает BTC, ETH, Total Cap
• Ждёт подтверждения от всех

Правило: "Тренд не подтверждён пока BTC и ETH 
         не покажут одно направление"
```

### Схема коммуникации:
```
         ORCHESTRATOR
              │
   ┌──────────┼──────────┐
   │          │          │
   ▼          ▼          ▼
 Market    Bull       Bear
 Analyst  Researcher  Researcher
   │          │          │
   └──────────┼──────────┘
              │
   ┌──────────┼──────────┐
   │          │          │
   ▼          ▼          ▼
  SMC      Monte      Gann
 Agent    Carlo      Agent
   │          │          │
   └──────────┼──────────┘
              │
              ▼
         SYNTHESIZER
              │
              ▼
         ПОЛЬЗОВАТЕЛЬ
```

### Правила:
```
1. ВСЕГДА только через Orchestrator
   ❌ Агент A НЕ может позвонить Агенту B
   
2. Строгий формат ответов:
   {
     "signal": "BUY/SELL/HOLD",
     "confidence": 0-100,
     "summary": "1-2 предложения",
     "details": {...}
   }

3. Timeout — никто не ждёт вечно
   • Market Analyst: 120 сек
   • Monte Carlo: 30 сек
```

## 🛡️ Конституция Агентов

### Что КАЖДЫЙ ОБЯЗАН:
```
1. ОТВЕЧАТЬ ВСЕГДА
   "HOLD, 30%" лучше чем молчание

2. УКАЗЫВАТЬ УВЕРЕННОСТЬ
   Нельзя: "Я думаю... может быть..."
   Можно: "Сигнал BUY, уверенность 75%"

3. ДАВАТЬ STOP LOSS
   Если BUY — должен сказать где стоп

4. ПРИЗНАВАТЬ ОГРАНИЧЕНИЯ
   Сказать если данных мало
```

### Что НЕ МОЖЕТ:
```
❌ Торговать сам
❌ Получить доступ к деньгам
❌ Менять настройки других
❌ Врать о данных
❌ Игнорировать плохие новости
```
