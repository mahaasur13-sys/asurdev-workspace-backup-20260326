## 8. Belief Tracker (Bayesian)

**Built:** 2026-03-26 ✅

### File: `core/belief.py` + `core/belief.db`

| Table | Purpose |
|-------|---------|
| `agent_beliefs` | Current Beta(α, β) per agent |
| `agent_belief_history` | Per-session outcomes (last 100 per agent) |

### Model

```
α = successes + 1   (default: α=1, β=1 → uniform prior)
β = failures  + 1

mean = α / (α + β)          — posterior mean accuracy
mode = (α-1) / (α+β-2)     — MAP estimate
std  = √(αβ / (n²(n+1)))   — posterior std dev
CI   = Wilson score interval (95%)
```

### Success Criteria

```
is_success = (final_signal == agent_signal)
             AND final_signal ∈ {LONG, SHORT, BUY, SELL, STRONG_BUY, STRONG_SELL}
```

### API

```python
from core.belief import get_belief_tracker, update_beliefs_from_session

# Auto-called after every run_sentinel_v5() (persist=True)
update_beliefs_from_session(final_output)

# Manual
bt = get_belief_tracker()
bt.leaderboard()                      # ranked by accuracy
bt.get_agent_history("QuantAgent")    # last 100 outcomes
bt.reset("QuantAgent")                # reset one / all
```

## 9. Thompson Sampling (Agent Selection)

**Built:** 2026-03-26 ✅

### Files

| File | Purpose |
|------|---------|
| `core/thompson.py` | `ThompsonSampler` + `AgentPool` |
| `orchestration/sentinel_v5.py` | Orchestrator with Thompson selection |
| `agents/astro_council_agent.py` | Filters sub-agents via `_thompson_selected_astro` |
| `tools/thompson_cli.py` | CLI: scores, select, leaderboard, simulate, reset |

### Agent Pools

| Pool | Agents | min | max |
|------|--------|-----|-----|
| `technical` | MarketAnalyst, BullResearcher, BearResearcher | 1 | 3 |
| `electoral` | ElectoralAgent | 1 | 1 |
| `astro` | FundamentalAgent, MacroAgent, QuantAgent, OptionsFlowAgent, SentimentAgent, TechnicalAgent, BullResearcher, BearResearcher | 3 | ∞ |

### Algorithm

```
For each agent i in pool:
    α_i, β_i = belief.get(agent_name)   # from belief.db
    if unseen: α_i, β_i = 1, 1          # uniform prior
    θ_i = sample Beta(α_i, β_i)         # NumPy np.random.beta

Sort agents by θ_i descending
Select top-K agents (or pool.min_select if larger)
```

### Selection Order (dependency-aware)

```
technical → (excluded) → astro → electoral
```

- BullResearcher / BearResearcher excluded from `astro` pool if already in `technical`
- `select_with_exclusions` creates a temporary pool copy (no mutation)

### CLI

```bash
python tools/thompson_cli.py scores --pool astro
python tools/thompson_cli.py select --pool astro --k 4
python tools/thompson_cli.py leaderboard
python tools/thompson_cli.py simulate --pool astro --k 4 --n 100 --seed 42
python tools/thompson_cli.py reset --agent QuantAgent
```

### Thompson vs Static Weights

| Aspect | Static AGENT_WEIGHTS | Thompson Sampling |
|--------|---------------------|-------------------|
| Selection | Fixed, all agents always called | Dynamic, top-K per flow |
| Exploration | None | Unseen agents equally likely |
| Exploitation | 100% static | Bayesian posterior → sample |
| Determinism | Fully deterministic | Stochastic (seedable) |

### Key Decisions

- **NumPy `np.random.beta`** — clean, battle-tested Beta sampler
- **Belief state** from `belief.db` — no duplication between systems
- **Backward compatible** — if `_thompson_selected_astro` absent, AstroCouncil runs all
- **`flows_run` unchanged** — sub-agent selection stored in `thompson_selections` field

## 10. LangGraph Belief-Guided Architecture

**Built:** 2026-03-26 ✅

### File: `langgraph_schema.py` (rewritten)

Replaces the old hard-coded sequential graph with a **Belief-Guided conditional graph**.

### Core Idea

Each graph node **self-decides** whether to run using Thompson Sampling — no central orchestrator pre-selects agents before the graph starts.

```
┌─────────────────────────────────────────────────────────────────┐
│  START                                                           │
│  │                                                              │
│  ├─→ technical_node()  ──→ [BeliefGuard: skip or run?]        │
│  │                           │                                  │
│  │                    ┌──────┴──────┐                          │
│  │                    ↓             ↓                          │
│  │               [run 2-3]      [SKIP → next]                   │
│  │                    │                                          │
│  ├─→ astro_council_node() ──→ [BeliefGuard: skip or run?]     │
│  │                           │                                  │
│  │                    ┌──────┴──────┐                          │
│  │                    ↓             ↓                          │
│  │               [run 3-8]       [SKIP → next]                  │
│  │                    │                                          │
│  ├─→ electoral_node() ──→ [BeliefGuard: skip or run?]           │
│  │                           │                                  │
│  │                    ┌──────┴──────┐                          │
│  │                    ↓             ↓                          │
│  │               [run 1]        [SKIP → next]                   │
│  │                    │                                          │
│  └─→ synthesis_node() ← collects signals from ALL flows        │
│                                                                  │
│  END                                                             │
└─────────────────────────────────────────────────────────────────┘
```

### How BeliefGuard Works

**`_pool_decide(pool, k_override)`** — called at the start of every node:

```
1. For each agent in pool:
       belief = belief_tracker.get(agent_name)
       if belief and belief.mean < pool.min_usefulness:
           skip agent (below threshold)
       else:
           α, β = belief or (1, 1)   # unseen = uniform prior
           θ = BeliefTracker._sample_beta(α, β)   # NumPy Beta sample
           eligible.append((name, θ))

2. If ALL below threshold → fallback to all agents (must have something)

3. Sort eligible by θ descending, take top-K

4. Return (should_run, selected_agents)
   - should_run = len(selected) >= pool.min_select
   - If False → node returns state unchanged, routes directly to next node
```

### Pool Thresholds

| Pool | `min_usefulness` | `min_select` | Behaviour |
|------|-----------------|--------------|-----------|
| `technical` | 0.25 | 1 | Low — small pool, don't filter aggressively |
| `electoral` | 0.30 | 1 | Always 1 agent, threshold for consistency |
| `astro` | 0.30 | 3 | 8 agents, threshold filters weak performers |

### Thompson Sampling Inside vs Outside Graph

| Aspect | `orchestration/sentinel_v5.py` | `langgraph_schema.py` |
|--------|-------------------------------|----------------------|
| Selection timing | **Before** graph runs | **Inside** each node |
| Node skip | Via conditional edge routing | Via BeliefGuard at node start |
| `synthesis` always runs | Yes | Yes |
| Flow order | Parallel (all at once) | Sequential (tech → astro → electoral → synth) |
| Use case | Fast batch runs | Explainable step-by-step trace |

### What Gets Stored in State

```python
state["thompson_selections"] = {
    "technical": ["MarketAnalyst", "BullResearcher"],   # or None if skipped
    "astro":     ["QuantAgent", "FundamentalAgent", ...], # or None if skipped
    "electoral": ["ElectoralAgent"],                    # or None if skipped
}
state["technical_result"] = None          # if skipped
state["technical_result"] = {...}         # if run
```

### Logs Example

```
[BeliefGuard] 'technical' — filtered (low utility): ['BearResearcher'] | selected: ['MarketAnalyst']
[Graph] technical — selected: ['MarketAnalyst']
[Graph] astro_council — selected: ['QuantAgent', 'FundamentalAgent', 'SentimentAgent']
[Graph] electoral — selected: ['ElectoralAgent']
```

### Running the Graph

```python
from langgraph_schema import graph, AgentState

state = AgentState(
    symbol="BTCUSDT",
    timeframe="SWING",
    current_price=97000,
    include_technical=True,
    include_astro=True,
    include_electoral=False,
)

result = graph.invoke(state)
print(result["final_recommendation"])
```

### Conditional Routing (unchanged)

- `route_after_start` → first active node based on `include_*` flags
- `route_from_technical` → skip directly to next active node (BeliefGuard handles skip)
- `route_from_astro` → skip directly to next active node

> **Note:** Skipped nodes set their result to `None` and let the conditional edge routing advance to the next node. Synthesis always runs last and collects whatever signals were produced.

### Changes from Old Graph

| Aspect | Old (`langgraph_schema.py`) | New |
|--------|---------------------------|-----|
| Agent selection | All agents always called | Thompson-sampled per pool |
| Node skip logic | Only via `include_*` flags | Both `include_*` flags AND BeliefGuard |
| AstroCouncil | Runs all 8 sub-agents | Thompson-selected subset via `_thompson_selected_astro` |
| Synthesis | Sequential after all | Always last, collects from all active flows |
| Parallelism | All flows at once | Sequential (tech → astro → electoral → synth) |

## 11. Agent Selection Logging

**Built:** 2026-03-26 ✅

### File: `core/belief.py` + `migrations/0007_agent_selection_log.sql`

### New Table: `agent_selection_log`

```sql
CREATE TABLE agent_selection_log (
    session_id      TEXT NOT NULL,
    agent_name      TEXT NOT NULL,
    pool_name       TEXT NOT NULL,
    was_called      INTEGER NOT NULL CHECK (was_called IN (0, 1)),
    success_flag    INTEGER,   -- NULL if was_called=0, 0/1 if was_called=1
    created_at      TEXT NOT NULL DEFAULT (datetime('now')),
    PRIMARY KEY (session_id, agent_name)
);
```

### Pool Inference Map

| Pool | Agents |
|------|--------|
| `technical` | MarketAnalyst, BullResearcher, BearResearcher |
| `electoral` | ElectoralAgent |
| `astro` | FundamentalAgent, MacroAgent, QuantAgent, OptionsFlowAgent, SentimentAgent, TechnicalAgent |

> Unknown agents default to `astro` pool.

### How It Works

`update_from_session()` now calls `_log_session_selections()` after updating beliefs:

```
Called agents:   was_called=1, success_flag=0/1  (success if signal matched final)
Not-called:   was_called=0, success_flag=NULL
```

For every known-pool agent in the system — whether it produced a signal or was silently skipped by Thompson Sampling — a row is written to `agent_selection_log`.

### API

```python
from core.belief import get_belief_tracker

bt = get_belief_tracker()

# All agents called in session X
bt.get_selection_log(session_id="session_abc123")

# Last 100 calls for one agent
bt.get_selection_log(agent_name="QuantAgent", limit=100)

# Last 100 global log entries
bt.get_selection_log(limit=100)
```

### What the Log Enables

| Question | Query |
|----------|-------|
| "Which agents was QuantAgent out-competed by?" | `WHERE agent_name="QuantAgent" AND was_called=0` |
| "How often does BullResearcher win selection?" | `WHERE agent_name="BullResearcher" AND was_called=1` |
| "Selection win rate per agent (called sessions only)" | `SELECT agent_name, AVG(success_flag) WHERE was_called=1` |
| "Thompson bias: agents never selected?" | `SELECT agent_name, COUNT(*) WHERE was_called=0 GROUP BY agent_name` |

### Belief Update Still Primary

`update_beliefs_from_session()` and `_update_agent()` are unchanged — they only update Beta(α, β) for agents that produced a signal. The selection log adds an **orthogonal** tracking dimension: whether an agent was even selected, regardless of whether it produced a signal.

## 12. Thompson Hyperparameters

**Built:** 2026-03-26 ✅

### Files

| File | Changes |
|------|---------|
| `core/thompson.py` | `AgentPool.k`, `exploration_bonus` param, K resolution logic |
| `langgraph_schema.py` | `_pool_decide` uses `sampler.exploration_bonus` |
| `tools/thompson_cli.py` | `--k`, `--exploration-bonus`, `--seed` flags |

### Three Hyperparameters

#### K — agents to select per pool

| Pool | K source | Result |
|------|----------|--------|
| `technical` | `pool.k=None` → `default_k=4` | 3 agents in pool, all can be selected |
| `electoral` | `pool.k=1` | Always 1 |
| `astro` | `pool.k=None` → `default_k=4` | 8 agents, selects 4 |

**Resolution order:** `k_override` argument > `pool.k` > `sampler.default_k`

```python
# Override pool.k in code
pool = AgentPool(name="astro", agents=[...], k=3)  # always select 3

# Or per-call
sampler.select(pool, k=5)  # override even pool.k=3
```

#### min_success_prob — threshold (already existed)

`pool.min_usefulness` (default 0.30) — agents with `mean_accuracy < 0.30` are excluded from sampling.

#### exploration_bonus — unseen agent prior boost

| Value | Prior for unseen | Effect |
|-------|-----------------|--------|
| `0.0` (default) | `Beta(1, 1)` uniform | Conservative — unseen treated as average |
| `1.0` | `Beta(2, 1)` optimistic | Unseen sampled ~66% mean — tries them more often |
| `2.0` | `Beta(3, 1)` very optimistic | Unseen sampled ~75% mean — aggressive exploration |

```python
# Conservative (default — same as before)
sampler = ThompsonSampler(exploration_bonus=0.0)

# Aggressive exploration — new agents tried frequently
sampler = ThompsonSampler(exploration_bonus=2.0)
```

### CLI Usage

```bash
# See scores with exploration_bonus=1.0 (unseen get Beta(2,1))
python tools/thompson_cli.py scores --pool astro --exploration-bonus 1.0

# Select 3 agents from astro pool (overrides default_k=4)
python tools/thompson_cli.py select --pool astro --k 3 --exploration-bonus 1.0

# Simulate 100 runs with aggressive exploration
python tools/thompson_cli.py simulate --pool astro --k 3 --n 100 --exploration-bonus 2.0 --seed 42
```

### Simulation: bonus=1.0 vs bonus=0.0 (k=3, 50 runs, seed=42)

| Agent | bonus=1.0 | bonus=0.0 |
|-------|-----------|-----------|
| FundamentalAgent | 60% | 44% |
| BullResearcher | 54% | 38% |
| OptionsFlowAgent | 46% | 26% |
| QuantAgent | 42% | 50% |
| SentimentAgent | 12% | 28% |

Higher `exploration_bonus` shifts selection frequency toward agents with no prior data, spreading load more evenly during early sessions.

### Recommended Tuning Workflow

```
1. Start: bonus=0.0 (conservative, measure baseline)
2. After 20 sessions: inspect belief leaderboard
   - If some agents have 0 sessions → raise bonus to 1.0 or 2.0
   - If all agents have >5 sessions → keep bonus=0.0
3. Tune K:
   - k=3: faster, less diverse signals
   - k=5: slower, more robust synthesis
   - k=8: all agents (no Thompson filtering — baseline comparison)
```
